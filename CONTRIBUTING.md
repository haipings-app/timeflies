# Contributing

Thanks for your interest in improving Time Left.

## Development Principles

- Keep the app focused on deadline visibility, actionable tasks, and reminders.
- Prefer native macOS APIs and low-dependency solutions.
- Keep features small and reviewable.

## Local Setup

1. Install a full version of Xcode.
2. Open the project folder in Xcode or build with SwiftPM once the local toolchain matches the active SDK.
3. Run tests before opening a pull request.

## Pull Requests

- Describe the user-facing change clearly.
- Include screenshots for UI changes when possible.
- Call out any reminder, persistence, or migration impact.
- Keep PRs scoped to one change area.

## Issues

- Use feature requests for new ideas.
- Use bug reports for regressions, crashes, or reminder failures.
